本文比较现代普通话、现代日语与现代英语在对话、写作、音节、文字、跨语言转换和新概念构造中的不同“密度”，并追问这些差异能否支持稳定的语言优劣或“智能密度”排序。

本次修订逐项检验 Reddit 评论中“字母表紧凑度—词语紧凑度—句子紧凑度”的区分，并新增 Unicode、字素簇、字体宽度、终端列宽、UTF-8 字节及同语言转写实验。

## 摘要

“哪种语言的信息密度最高”不是一个单一问题。结论会随计量单位而改变：按**可见字符**计，中文在许多同内容正式文本中通常最短；按 **UTF-8 字节**计，这种优势会大幅缩小，甚至被工程编码方式逆转；按**音节所承载的信息**计，普通话和英语在经典实验中高于日语；按**每秒信息率**计，日语通过更高的音节速率进行补偿，跨语言差异显著收敛；按大语言模型的 **token** 计，结果主要受分词器、训练语料和词表设计影响，不能被解释为语言本身的固有属性。

本修订所检验的 Reddit 评论抓住了一个必要的方法论区分：**书写符号序列短，不等于词汇表达短；词汇表达短，也不等于整句、话语、页面或数字编码更紧凑。** 但评论使用“字母表”统称汉字、假名和拉丁字母，并把日语假名近似为“一个符号等于两个英文字母”、把汉字近似为“一个符号等于一个完整单词”、把英语名词串归因于“自由语序”，这些解释均不够准确。更合适的术语是**文字系统粒度**：基本图形单位对应音位、拍、音节、语素还是词，以及该单位是否同时编码语音和意义。[52][53]

新增的受控计数显示，联合国《世界人权宣言》第一条中，日文原文只有英文一半的 Unicode code point，但在同一 Noto Sans CJK 字体下，其估计水平行进宽度略大于英文；在典型终端列宽规则下，两者均占 170 列。中文仍明显较短，但其行进宽度约为英文的 52%，而不是 code point 数所暗示的 25%。同一普通话内容改写为汉语拼音、同一日语内容改写为全假名或罗马字，也会在不更换语言的情况下大幅改变字符数和字节数。这证明“语言紧凑度”必须与“文字系统和数字表示紧凑度”分开。

Pellegrino、Coupé 与 Marsico（2011）在七种目标语言的平行朗读材料中发现，语言之间存在“单位信息量—发音速度”的负相关：日语每音节信息量低但音节速率高，普通话每音节信息量高但速率低；不过其数据并不支持所有语言具有严格相同的信息率。[3] Coupé 等（2019）将样本扩大到 17 种语言、170 名说话人，以音节熵计算信息量，得到跨语言平均约 **39.15 bit/s**、标准差约 **5.10 bit/s**，支持“相近的编码效率”，而非一个精确的语言普适常数。[4]

本文同时否定“智能密度”作为语言固有等级指标。该词不是语言学、心理测量学或信息论中的标准量。高香农熵可能来自随机噪声；字符少可能依赖更多背景推理；命题多可能增加理解成本；计算 token 少可能只是分词器更偏爱该语言。因此，较可用的替代概念是**任务条件下的交际效率**：在给定语境、错误容忍度、传输时间和认知成本时，接收者成功恢复多少目标意义。

本文的主要结论是：

1. **不存在不依赖单位、语料和任务的中—日—英总体密度排序。**
2. **中文的视觉书写紧凑性是真实但局部的现象**，主要来自形态音节文字、无空格书写、低屈折和复合构词；它不等同于更高的每秒信息率或更高认知效率。
3. **日语在控制朗读中通常每音节信息较少、音节速率较高**；“日语冗长”只在某些表面单位上成立，不能忽略拍、词法屈折、助词、礼貌与社会关系信息。
4. **英语并不依赖不断创造不可分析的新原子词。** 英语、中文、日文都能通过复合、派生、转类、缩略、借词和仿译扩展概念系统；差别是构词资源、词汇化偏好和书写惯例，而非“可组合/不可组合”的二分。
5. **对话与写作必须分开。** 对话依赖共享语境、指示、零成分、修正和轮替；写作则通常增加名词化、显式衔接和可脱离现场的解释。词面密度低的对话可以具有很高的互动效率。
6. **认知实验未证明某种语言具有更高的“智能”。** 眼动范围、短时记忆广度和阅读脑网络的差异，主要反映书写系统、发音时长、切分线索、熟练度和任务策略。
7. **工程成本不是语言本体。** UTF-8、压缩算法和神经网络 tokenization 都会改变表面长度；它们应作为系统设计变量报告，而不是用来给语言排序。


## 0. 对 Reddit 观点的逐项判定

### 0.1 总体结论 {.no-toc}

该评论的**核心区分正确，具体机制大多错误或过度简化**。它正确地指出：符号库存、书面词形、整句表达和口头话语不是同一层级；但它随后又把这些层级混在一起，用文字类型直接推出句子长度和语言效率。[50]

| Reddit 命题 | 判定 | 更准确的说明 |
|---|---|---|
| 应区分“字母表紧凑度”、词语紧凑度和句子紧凑度 | **正确，但术语需改** | 应称“文字系统粒度/正字法单位”，因为中文不是字母文字，标准日文也不是单一音节文字 |
| 符号库存中每个符号包含更多信息，就会减少每页符号数 | **有条件成立** | 较粗粒度单位可减少符号序列长度；但页面面积还受字形宽度、高度、字体、行距和可辨识尺寸影响 |
| 一个日语符号大致等于两个英文字母 | **只是一种很粗的罗马字直觉** | 平假名、片假名主要对应**拍**；`か/ka` 是 1:2，`あ/a` 和 `ん/n` 是 1:1，`きゃ/kya` 是 2:3；标准日文还大量使用汉字[53] |
| 一个汉字位置可以是一个完整单词 | **有时成立，不能一般化** | 汉字通常是形态音节或语素—音节单位；有些字可独立成词，但现代汉语大量词由两个或更多汉字构成[28][34][52] |
| 因此书写紧凑度普遍是“中文 < 日文 < 英文” | **不是普适定律** | 该排序常见于同内容的 code point 数，但可在 UTF-8 字节、字体宽度、终端列宽、词数或不同文体中缩小、消失或改变 |
| Google Translate 中西班牙语总比简洁英语长 | **不能作为证据** | 机器翻译输出受模型、翻译方向、术语和显化策略影响；一条输出不能估计语言总体长度 |
| 英语可以堆叠名词，是因为语序更自由 | **机制错误** | 现代英语基本语序相对固定；这里涉及的是名词—名词复合和名词前置修饰，而不是自由语序[59][60] |
| 英语名词串不需要额外结构 | **错误** | 结构和语义关系仍存在，只是常不由介词显式标出；关系越隐含，歧义和推理负担可能越高[59] |

### 0.2 “字母表”不是这里的共同上位概念 {.no-toc}

严格说，`alphabet` 指字母文字中的符号系统，典型单位与音位存在系统对应。中文、日文、英文的共同上位概念应是**书写系统**或**文字系统**：

- 英文主要使用字母文字；基本图形单位是字母，通常编码音位级线索；
- 日文是汉字、平假名、片假名、拉丁字母和数字的混合系统；假名主要是拍文字；
- 中文主要使用形态音节文字；汉字通常同时关联语素和音节，而不是纯粹表示“观念”。

Ping Chen 提出的两个比较维度更精确：第一，基本图形单位对应多大的语音片段；第二，该单位只编码声音，还是同时编码声音与意义。[52] 这两个维度比“哪个字母表更高级或更紧凑”更适合跨文字体系比较。

### 0.3 必须分开的映射链 {.no-toc}

同一条消息至少经过以下层级：

\[
\text{目标意义 }M
\rightarrow \text{语言形式 }L
\rightarrow \text{正字法单位序列 }O
\rightarrow \text{Unicode code point 序列 }U
\rightarrow \text{编码字节 }B
\rightarrow \text{字形与版面 }A
\]

- 从 \(M\) 到 \(L\)：由词汇、形态、句法、语用和省略规则决定；
- 从 \(L\) 到 \(O\)：由汉字、假名、拉丁字母、空格和标点惯例决定；
- 从 \(O\) 到 \(U\)：由 Unicode 字符选择和规范化形式决定；
- 从 \(U\) 到 \(B\)：由 UTF-8、UTF-16、UTF-32 或其他编码决定；
- 从 \(U\) 到 \(A\)：由字体、字形替换、字号、字距、行距、书写方向和排版宽度决定。

Reddit 评论的第一句实际上要求分开这些映射；后续解释却把 \(O\)、\(L\) 和 \(A\) 再次合并。修订后的核心命题应是：

> **较粗粒度的书写单位往往减少同一语言形式所需的图形单位数，但不会单独决定词数、句长、页面面积、传输时间、字节数或理解成本。**

### 0.4 “紧凑”不等于“高效” {.no-toc}

省去显式关系标记可缩短形式，却可能增加歧义。增加重复和确认会拉长对话，却可能降低通信错误。较复杂字形可减少序列单位，却可能要求更大的可辨识尺寸。因而至少要同时观察：

\[
D_{\text{sequence}}=\frac{I(M;\hat M)}{N_{\text{unit}}},\qquad
D_{\text{area}}=\frac{I(M;\hat M)}{A_{\text{rendered}}},\qquad
D_{\text{byte}}=\frac{I(M;\hat M)}{B},\qquad
D_{\text{time}}=\frac{I(M;\hat M)}{T}
\]

四个分母分别是序列单位、实际版面面积、编码字节和时间。它们完全可能给出不同排序。


## 1. 定义与理论基础

### 1.1 “信息密度”至少有六种不同含义 {.no-toc}

| 层级 | 典型指标 | 计量单位 | 它实际回答的问题 | 主要缺陷 |
|---|---:|---|---|---|
| 视觉/排版密度 | 字符数、字形数、行宽 | 字符/句、字形/面积 | 同一文本占多少版面 | 字符承载量跨文字体系不等；字体与排版会改变结果 |
| 语言结构密度 | 词、语素、音节、拍 | 语义单位/表面单位 | 多少形式单位表达一组内容 | “词”和“语素”切分跨语言不统一 |
| 统计信息密度 | surprisal、熵、条件熵 | bit/符号 | 当前符号消除了多少统计不确定性 | 高熵不等于高语义价值；强依赖语料和模型 |
| 时间信息率 | 信息量 × 产生速率 | bit/s、命题/min | 单位时间传输多少可预测性降低 | 朗读速度不等于自然对话速度；单位定义影响结果 |
| 语义/命题密度 | 命题数、idea density | 命题/词、命题/句 | 文本压入多少可识别命题 | 命题标注依赖理论与语言分析器；不直接反映可懂度 |
| 工程编码密度 | UTF-8 字节、压缩后字节、模型 token | byte/文本、token/文本 | 存储、计费、上下文窗口和计算成本 | 取决于编码与分词器，不是语言固有量 |

### 1.1.1 “文字系统紧凑度”还要拆成库存、序列和空间 {.no-toc}

| 量 | 形式化对象 | 示例问题 | 不能推出什么 |
|---|---|---|---|
| 符号库存规模 | 类型数 \(K\) | 常用基本符号有多少种 | 不能直接推出一段文本需要多少符号 |
| 符号序列长度 | token 数 \(N\) | 同一内容需要多少字母、假名或汉字 | 不能直接推出页面宽度或语义量 |
| 实际统计熵 | \(H(S_t\mid S_{<t})\) | 一个符号平均消除多少不确定性 | 不能等同于概念价值或理解率 |
| 空间占用 | 行进宽度、包围盒、面积、行数 | 同一字体和版心下占多少版面 | 不能由 code point 数单独推得 |
| 数字表示 | code unit、byte、压缩 byte | 存储和传输成本多大 | 不能作为语言本体的语义效率 |

若库存中有 \(K\) 个**独立且等概率**的符号，单符号最大容量是：

\[
H_{\max}=\log_2 K
\]

这说明较大的符号库存原则上可用较少符号编码同一组消息，类似十进制通常比二进制使用更少位数。但自然语言符号并非独立等概率：高频字、字母和假名分布高度偏斜，并受上下文强烈约束；大量罕见汉字不会让每个常用汉字自动获得 \(\log_2 K\) bit。更大的库存还把成本转移到学习、辨认、输入、字体和错误恢复上。

任何“最高密度”结论都必须先补全一个分母：**每个字符、每个音节、每秒、每个字节、每个 token，还是每单位认知成本？** 不补全分母，比较没有确定含义。

### 1.2 香农信息、惊奇度与熵率 {.no-toc}

若事件或符号 \(x\) 的概率为 \(p(x)\)，其惊奇度为：

\[
I(x)=-\log_2 p(x)
\]

随机变量 \(X\) 的熵为：

\[
H(X)=-\sum_x p(x)\log_2 p(x)
\]

自然语言存在上下文依赖，因此更有意义的是条件熵或熵率，例如：

\[
H(X_t\mid X_{<t})
\]

单位时间的信息率可写成：

\[
R=H_{\text{unit}}\times v_{\text{unit}}
\]

其中 \(H_{\text{unit}}\) 是每个单位的平均信息量，\(v_{\text{unit}}\) 是每秒产生的单位数。[1][2]

这一定义解释了为什么“单位密度高”与“总传输率高”不是同一件事：一种语言可以让每个音节承载较少信息，同时通过更快的音节速率补偿。

### 1.3 统计信息不等于语义信息 {.no-toc}

以下三个例子不能混同：

- 一串均匀随机字符可能有很高的香农熵，但没有可恢复的语义结构。
- 一条高度可预测的法律套语可能统计熵低，却承担明确的制度功能。
- 一句极短的诗可能字符少、歧义多，并要求读者进行大量推理；它在表面上“密”，但解码成本可能很高。

因此，“字符少”“不可预测”“命题多”“理解快”是四个不同变量。

### 1.4 “均匀信息密度”假说的边界 {.no-toc}

均匀信息密度（Uniform Information Density, UID）相关研究认为，说话者倾向于避免局部信息峰值，并通过可选词、缩减、时长和句法选择让信息分布较平滑。[5][6] 但后续综述指出，UID 有多个不完全等价的形式化版本；不同实验可能同时兼容“避免高峰值”“维持稳定率”或一般处理难度解释，不能把它直接升级为“人类语言具有一个固定信道容量”。[8]

跨 600 种语言的语音研究发现，较不可预测的音素通常持续更久，较可预测的音素通常更短；显著关系出现在 600 种语言中的 319 种，而不是所有语言。[7] 这支持广义的**惊奇度—时长权衡**，但不支持机械的普适常数。

### 1.5 对“智能密度”的严格处理 {.no-toc}

“智能密度”不是公认的语言学或认知科学指标。将它解释为“某语言或其使用者更聪明”是范畴错误：语言是编码系统和社会实践，不是智力测验分数。

若保留这个词，只能把它操作化为**任务条件下的意义恢复效率**。一个可讨论但尚未标准化的框架是：

\[
\eta_T=\frac{I(M;\hat M)}{T+\alpha E+\beta C+\gamma L}
\]

其中：

- \(M\)：发送者的目标信息；
- \(\hat M\)：接收者实际恢复的信息；
- \(I(M;\hat M)\)：目标信息与恢复信息的互信息；
- \(T\)：传输时间；
- \(E\)：错误风险或错误代价；
- \(C\)：编码与解码认知成本；
- \(L\)：对额外背景知识的依赖；
- \(\alpha,\beta,\gamma\)：任务给定的权重。

这不是一个已经验证的量表，而是一种防止概念混淆的分析框架。没有任务、受众和语境，所谓“智能密度”没有可比较值。


## 2. 结构差异与历史成因

### 2.1 先区分语言、文字和编码 {.no-toc}

中文、日文和英文的比较至少涉及三个层次：

1. **语言系统**：音系、词法、句法、语用；
2. **书写系统**：汉字、假名、拉丁字母及其组合；
3. **数字编码系统**：Unicode、UTF-8、压缩格式、模型 token。

“中文四个字比英文二十个字母短”首先是书写单位比较，不自动推出普通话音节更少、理解更快、存储字节更少或语义更多。

因此，本文不把“字母表紧凑度”作为总称，而使用三个更明确的术语：**文字单位粒度**、**正字法序列长度**和**版面空间占用**。前者讨论单位对应音位、拍、音节、语素还是词；第二个讨论一条消息需要多少单位；第三个才讨论每页或每屏实际容纳多少内容。

### 2.2 中文：形态音节文字、低屈折与复合构词 {.no-toc}

现代汉字通常同时提供语素和音节线索，但“一个字必然等于一个词、一个意义、一个音节”均不严格成立：多音字、语素虚化、多字词和异体/简繁关系都会破坏简单一一对应。[28][34]

普通话的若干相关特征是：[30]

- 动词、名词的屈折变化相对少；时态通常不通过动词词形强制标记，而由体标记、时间副词和语境表达；
- 语序、虚词、话题结构和语境承担较多句法/篇章功能；
- 现代词汇中双音节词和复合词占重要位置；历史音变造成同音语素增多、双音化和复合化有助于区分词项；
- 无空格连续书写使字符序列在视觉上紧凑，但也把“词切分”任务交给读者或算法；
- 声调扩大了可区分音节库存，因此按音节计算时，单音节可具有较高的统计辨别力。

“中文能用旧字组合新概念”是事实的一部分，但不是中文独有能力。`人工智能`、`基因编辑`、`云计算`、`量子计算`都是复合或仿译；同时中文也吸收音译、字母词和缩略语，如 `咖啡`、`雷达`、`DNA`、`AI`。[28][29]

### 2.3 日文：黏着形态、拍结构与混合文字 {.no-toc}

日语通常被描述为黏着程度较高、主要为 SOV/中心语后置的语言；助词标记格和话题，动词/形容词具有较系统的屈折，语境允许论元省略，敬语和礼貌体系可以编码参与者关系。[31][32]

其书写系统混合使用：

- 汉字：常提供词汇意义和视觉切分线索；
- 平假名：常用于屈折词尾、功能成分和本土词；
- 片假名：常用于外来词、拟声拟态、术语突出；
- 拉丁字母与数字：在技术和商业文本中常见。

日语语音节奏和韵律分析中，“拍”（mora）是关键单位。以音节统一比较跨语言时，日语内部的长音、促音、拨音等拍结构会被压缩进不同的“音节”定义；因此日语按音节或按拍计算可能得到不同密度。Kubozono 把平假名和片假名描述为拍本位系统；假名与拍高度对应，但拗音等可由两个假名字符组合表示一个拍。[53] 所以把假名简单称为“一个符号等于两个英文字母的音节文字”会同时混淆拍、音节、假名字符和罗马字。

日语词汇存在本土词、汉语词、外来词和拟声拟态词等多个词汇层。新概念既可用汉字复合词，如 `人工知能`，也可使用片假名借词，如 `クラウドコンピューティング`，还可缩写为 `AI`。其扩展能力不低于中文或英文，只是借词的显性程度更高。[31]

### 2.4 英文：字母文字、相对固定语序与高生产力构词 {.no-toc}

英语以拉丁字母为基础，正字法的字母—音位对应较不透明；空格显式标词，词形屈折比古英语减少，但名词数、动词时态/一致、冠词和代词仍承担若干强制或半强制语法功能。固定语序和功能词在语法关系表达中很重要。[33][35]

英语能够形成长名词串，但这不是“自由语序”的结果。现代英语陈述句的基本成分顺序相对固定；名词串属于名词复合和名词前置修饰的局部构式。其紧凑性来自把名词间关系留为隐含，而不是取消了句法结构。长名词复合也会给人类和自动分析造成结构与语义歧义。[59][60]

英文同样具有高度生产力的新概念构造机制：

- 复合：`cloud computing`, `gene editing`, `smartphone`；
- 派生：`digitize`, `decarbonization`, `multimodal`；
- 转类：`to Google`, `to message`, `a must`；
- 缩略：`AI`, `DNA`, `radar`；
- 借词与混成：`karaoke`, `emoji`, `biotech`。

英文并非“每遇新概念就必须创造一个不可分解的新单词”。英文的名词—名词复合是其最有生产力的构词方式之一。[33]

### 2.5 历史路径为什么会留下不同的“密度外观” {.no-toc}

三种语言今天的表面长度并非由单一“效率原则”设计出来，而是历史叠加的结果：

- **中文**长期以汉字记录语素—音节单位。历史音系合并增加了同音形式，双音化、复合化和语境约束共同承担消歧；现代简化字主要改变部分字形笔画，并未把语言变成更高熵的代码。[28][29][34]
- **日文**在吸收汉字后逐步形成假名，并把汉字词干、假名屈折和外来词片假名组合成混合系统。汉语词、本土词、拟声拟态词和近现代外来词的多层词汇来源，使同一概念常有不同脚本和语体选择。[31][32][34]
- **英文**经历屈折弱化、大量法语和拉丁语词汇进入、语音持续变化以及拼写规范较早固定。结果是现代英语语法较依赖语序和功能词，而拼写保留许多历史层，字母数与当代发音并不简单对应。[33][35]

这些历史过程解释的是“为什么成本被放在不同位置”：中文把较多区分压入字形和词汇组合，日文把区分分配到混合脚本、助词和屈折，英文把区分分配到字母串、空格、功能词和相对固定语序。它们不是从同一起点优化出的三种编码方案，因此不能只以一个表面单位裁决效率。

### 2.6 结构对密度的作用不是单向的 {.no-toc}

| 结构特征 | 可能降低表面长度 | 可能增加表面长度或成本 |
|---|---|---|
| 中文低屈折、论元可省 | 少写时态、数、一致和代词 | 依赖语境；消歧时需补充词语 |
| 汉字语素线索 | 少量字形可表示多音节概念 | 学习字形库存和词切分成本较高；UTF-8 常为多字节 |
| 日语论元省略 | 对话中可省主语、宾语 | 助词、屈折、礼貌形式和句尾成分增加形式长度 |
| 日语汉字—假名混写 | 汉字提供压缩和切分线索 | 同一词改写为假名时字符数上升；读音映射复杂 |
| 英语空格与字母拼写 | 词边界显式，检索和分词方便 | 单个语素常需多个字母；冠词、介词、代词较显式 |
| 英语复合与缩略 | 可快速构造透明术语 | 正字法可能分写为多个词，字符数看起来更长 |

因此，结构差异通常是在**显式性、紧凑性、可预测性、学习成本和歧义风险**之间重新分配代价，而不是免费增加信息。


## 3. 口语实证：每音节信息与每秒信息

### 3.1 Pellegrino、Coupé 与 Marsico（2011） {.no-toc}

#### 设计 {.no-toc}

研究使用 20 段原始英语文本，将其翻译为法语、德语、意大利语、日语、普通话和西班牙语；每段由约五个连续句子组成。七种目标语言共有 59 名说话人、585 段录音、约 150 分钟语料，另以越南语作为外部参照。[3]

该研究的“信息密度”不是直接估计香农熵，而是以平行文本表达同一语义内容所需的音节数为基础，相对于越南语归一化。其“信息率”是归一化信息密度与音节速率之积。

#### 三语结果 {.no-toc}

| 语言 | 归一化信息密度 ID | 音节速率 SR（音节/秒） | 归一化信息率 IR |
|---|---:|---:|---:|
| 英语 | 0.91 ± 0.04 | 6.19 ± 0.16 | 1.08 ± 0.08 |
| 日语 | 0.49 ± 0.02 | 7.84 ± 0.09 | 0.74 ± 0.06 |
| 普通话 | 0.94 ± 0.04 | 5.18 ± 0.15 | 0.94 ± 0.08 |
| 越南语参照 | 1.00 | 5.22 | 1.00 |

#### 可支持的结论 {.no-toc}

- 日语在该材料中每音节承载的相对语义量最低，但说得最快；
- 普通话每音节密度较高，但音节速率较慢；
- 英语密度较高且速度中等，因此该样本中的归一化信息率最高；
- 七种语言中有五种的信息率较集中，但英语和日语构成明显对照，因此不能说实验发现了“完全相同的信息率”。[3]

#### 不能过度推出的结论 {.no-toc}

- 研究对象是**控制朗读**，不是自然对话；
- 每种语言约 6—10 名说话人，语言数只有七种，不能代表全球语言；
- 音节跨语言并非完全同质单位，尤其对日语的拍结构；
- 翻译文本可能带有源语言结构和“翻译腔”；
- 结果显示说话人和文本本身也是显著变异来源。

### 3.2 Coupé 等（2019）：17 种语言的相近编码效率 {.no-toc}

#### 设计与指标 {.no-toc}

该研究扩展到 17 种语言、9 个语系、170 名母语说话人。每位说话人朗读最多 15 段语义平行短文。研究者先从各语言大型书面语料中估计音节概率，再计算：

- 一阶音节熵 \(ShE\)；
- 给定同词前一音节的二阶条件熵 \(ID\)；
- 朗读音节速率 \(SR=NS/Duration\)；
- 信息率 \(IR=ID\times SR\)。[4]

这里的 \(ID\) 只利用词内前一个音节的局部条件，不跨词建模，也不使用句法、篇章或共同知识。它比一阶熵更接近序列信息率，但仍不是自然语言的完整熵率；换用更长上下文或神经语言模型，绝对 bit 数会改变。

模型给出的跨语言平均信息率约为 **39.15 bit/s**，标准差约 **5.10 bit/s**；平均音节速率约 **6.63 音节/秒**，标准差约 **1.15**。[4]

#### 对作者公开数据的描述性复算 {.no-toc}

下表由论文附带的 `InfoRateData.csv` 复算。计算对每条录音等权，未使用论文的混合效应模型，因此只用于透明描述，不能替代论文的调整后推断。

| 语言 | 录音数 | 说话人数 | 音节熵 ShE（bit/音节） | 条件信息密度 ID（bit/音节） | 平均音节速率 SR | 平均信息率 IR（bit/s） | IR 标准差 |
|---|---:|---:|---:|---:|---:|---:|---:|
| 普通话（CMN） | 150 | 10 | 8.69 | 6.96 | 5.858 | 40.770 | 4.300 |
| 英语（ENG） | 60 | 10 | 9.51 | 7.09 | 6.338 | 44.938 | 4.778 |
| 日语（JPN） | 150 | 10 | 6.07 | 5.03 | 8.035 | 40.415 | 2.626 |

这组原始均值再现了同一结构：

- 日语的单位音节信息最低、音节速率最高；
- 普通话单位音节信息较高、速率较低；
- 三者按 bit/s 的差距比按 bit/音节小得多。

但英语录音数少于普通话和日语，且录音并非独立的“语言样本”；正式比较必须使用论文的层级模型。

#### 2011 与 2019 为什么不能直接视为重复实验 {.no-toc}

| 方面 | 2011 | 2019 |
|---|---|---|
| 单位信息定义 | 平行文本所需音节数的相对语义密度 | 从语言语料估计的音节条件熵 |
| 语言数 | 七种目标语言，另有越南语参照 | 17 种语言 |
| 主要结论 | 存在速度—密度权衡，但非严格等率 | 信息率围绕约 39 bit/s 收敛 |
| 共同限制 | 控制朗读、翻译文本、音节定义 | 控制朗读、翻译文本、局部条件模型 |

两者的方向一致，但测量对象不同。2011 更接近“同一语义内容需要多少音节”，2019 更接近“音节序列平均减少多少统计不确定性”。

### 3.3 对自然对话的外推必须收缩 {.no-toc}

自然对话会引入朗读语料没有的变量：

- 停顿、犹豫、重复和自我修正；
- 反馈词与应答，如“嗯”“对”“そう”“right”；
- 共享视觉场景和共同知识；
- 轮替压力；
- 语气、礼貌、身份关系；
- 话题突然切换和非完整句。

这些单位可能不增加命题内容，却增加同步、纠错和社会协调功能。把它们视为“无用冗余”会漏掉对话真正传输的一部分信息。[15][46][47]


## 4. 书面实证：文字系统、字符、版面、Unicode 与平行语料

### 4.1 先把“符号库存”“文本序列”和“页面面积”分开 {.no-toc}

Reddit 评论把“alphabet compactness”理解为“一个符号承载的单位更大，所以每页符号更少”。这只覆盖了书面比较的一部分。至少应分开：

| 层级 | 对象 | 可用指标 | 典型决定因素 |
|---|---|---|---|
| 库存紧凑度 | 可用符号类型 | 类型数、有效词表大小 | 文字历史、规范、教育用字范围 |
| 序列紧凑度 | 一条文本中的符号 token | code point、字素簇、字母/假名/汉字数 | 文字单位粒度、拼写、空格、标点 |
| 词汇紧凑度 | 一个词或术语的形式长度 | 字符/词、音节/词、语素/词 | 复合、派生、借词、缩写、词汇化 |
| 句法紧凑度 | 一组命题的表面形式 | 词/命题、语素/命题、依存关系 | 屈折、功能词、语序、省略、信息结构 |
| 空间紧凑度 | 实际渲染后的版面 | em、像素、面积、行数、页数 | 字体、字号、全角/比例宽度、行距、断行 |
| 数字紧凑度 | 机器中的表示 | code unit、UTF-8 byte、压缩 byte、token | 编码格式、规范化、压缩器、分词器 |
| 认知紧凑度 | 成功恢复的意义 | 正确命题/秒、理解率/注视时间 | 熟练度、歧义、视觉复杂度、背景知识 |

这七种紧凑度并不保持同一排序。把“每页字符数”当作“实际句子紧凑度”仍然少了一步：页面容纳量取决于每个字形的物理宽度和高度，而不是只取决于序列中有多少 code point。

### 4.2 大符号库存可以缩短序列，但没有“免费信息” {.no-toc}

若一个书写系统有 \(K\) 个等概率独立符号，则每个符号的理论上限为 \(\log_2K\) bit。用十进制写 `255` 比用二进制写 `11111111` 短，是因为十进制单个数字来自更大的库存。

但把这一比喻直接套到汉字、假名和字母会产生四个问题：

1. 自然语言符号概率极不均匀，实际熵远小于 \(\log_2K\)；
2. 符号之间有强上下文依赖，必须估计条件熵；
3. 三种文字的符号映射到不同语言单位，不只是用不同“进制”记录同一底层串；
4. 大库存增加学习、辨认、输入、排错和字体覆盖成本。

所以“汉字库存大，因此每个汉字包含更多信息”只能作为待测假说。可测量的是给定语料和模型下的：

\[
H(C_t\mid C_{<t})
\]

而不是从字典收录多少字符直接推出每字信息量。

### 4.3 汉字是语素—音节单位，不是普遍的“整词格子” {.no-toc}

Wikipedia 的 `Logogram` 页面把 logogram 定义为表示词或语素等语义成分的书写字符，并指出历史上的语素文字都包含语音维度；该页面本身同时显示“需要更多引文”和“需要改写”的质量提示，因此可作术语入口，不应取代文字学专著。[51]

对现代汉语，更稳妥的描述是：汉字通常是**形态音节/语素音节**单位，一个字往往对应一个音节和一个语素，但并非严格一一对应。[28][34][52]

- `人` 在许多语境中可同时是一个字、一个语素和一个词；
- `自由` 是两个字构成的一个常用词；
- `葡萄` 是两个字构成的词，不能把每个字都当作该词中的独立现代词义；
- `的`、`了` 等主要承担语法功能，不能用“一个完整概念词”概括；
- 多音字、异读、简繁异体、专名和借词会进一步破坏简单映射。

因此，“一个汉字可以独立成词”是真的；“每个汉字位置都是一个完整单词”是假的。中文字符序列经常短，主要因为基本单位粒度较粗、空格不标词、屈折较少和复合构词活跃，而不是每个字都等于英文一个词。

### 4.4 日语假名主要对应拍，不等于固定两个拉丁字母 {.no-toc}

Kubozono 对日语韵律的综述指出，平假名和片假名按拍组织，假名与拍之间高度对应。[53] 这不等于“每个 Unicode code point 必然对应一拍”：多数拍写作一个假名，拗音等则由常规大小假名与小写假名组合表示。“一个假名约等于两个英文字母”只对一部分 CV 拍的罗马字转写看起来成立：

| 日语形式 | 罗马字 | 假名数 | 拉丁字母数 | 说明 |
|---|---|---:|---:|---|
| `か` | `ka` | 1 | 2 | 符合 Reddit 的粗略直觉 |
| `あ` | `a` | 1 | 1 | 元音拍不是 1:2 |
| `ん` | `n` | 1 | 1 | 拨音是一拍 |
| `きゃ` | `kya` | 2 | 3 | 常规大小的 `き` 加小写 `ゃ` 共同表示一拍 |
| `が` | `ga` | 1 个可见单位 | 2 | Unicode 可用一个预组 code point，亦可用两个 code point 表示 |

更重要的是，标准日文不是纯假名文本。汉字常编码词汇词干，假名常编码屈折、助词和其他功能成分。`情報密度` 与 `じょうほうみつど` 是同一日语词组的两种书写，前者 4 个汉字，后者 8 个假名；语言没有改变，正字法序列已经改变。

### 4.5 词语紧凑度与句子紧凑度由不同机制产生 {.no-toc}

“单词紧凑”至少有三种含义：

- **正字法词长**：一个以空格或约定边界划分的词有多少字符；
- **形态密度**：一个词包含多少语素和语法特征；
- **词汇化密度**：共同体是否已有一个短而固定的术语。

跨语言的“词数”尤其不稳定。英文 `machine learning` 按空格是两个正字法词；中文 `机器学习` 通常不写空格；日文 `機械学習` 也是连续书写。三者的词边界、复合分析和字符数不在同一尺度。

整句紧凑度还受以下因素影响：

- 一种语言把时态、数、性、格或敬语编码为屈折，另一种用独立功能词或语境；
- 论元能否省略，以及省略需要多少共同知识；
- 名词间语义关系是否用介词显式标出；
- 翻译是否为了法律精确、礼貌或消歧而显化信息；
- 形式缩短后，歧义和推理负担是否增加。

因此，短名词串可能只是把关系编码从表面形式转移到读者推理。它在字符层面更短，不必然在认知层面更省。

### 4.6 同一语言改换文字：把“语言”与“书写系统”隔离开 {.no-toc}

以下对照尽量保持同一语言、同一内容，只改变书写方式。计数前采用 NFC；“终端列”按 Unicode East Asian Width 的常见 `wcwidth` 规则估计；“行进宽度”使用同一套 Noto Sans CJK Regular 字体、100 px 字号，以字体 advance 除以 100 得到 em 近似值。

> **普通话汉字**：人人生而自由，在尊严和权利上一律平等。<br>
> **汉语拼音**：Rénrén shēng ér zìyóu, zài zūnyán hé quánlì shàng yīlǜ píngděng.

> **日文汉字假名混写**：すべての人間は、生まれながらにして自由であり、かつ、尊厳と権利とについて平等である。<br>
> **日文全假名**：すべてのにんげんは、うまれながらにしてじゆうであり、かつ、そんげんとけんりとについてびょうどうである。<br>
> **日文罗马字**：Subete no ningen wa, umarenagara ni shite jiyū de ari, katsu, songen to kenri to ni tsuite byōdō de aru.

| 语言与写法 | code point | 扩展字素簇 | UTF-8 byte | 终端列 | 字体行进宽度（em） |
|---|---:|---:|---:|---:|---:|
| 普通话：汉字 | 19 | 19 | 57 | 38 | 19.00 |
| 普通话：拼音 | 64 | 64 | 81 | 64 | 30.01 |
| 日语：汉字假名混写 | 42 | 42 | 126 | 84 | 41.73 |
| 日语：全假名 | 51 | 51 | 153 | 102 | 50.73 |
| 日语：罗马字 | 104 | 104 | 107 | 104 | 47.32 |

结果说明：

- 普通话汉字写法的 code point 只有拼音的 29.7%，但其物理行进宽度约为拼音的 63.3%；
- 日文混写比全假名少 17.6% code point，行进宽度也少约 17.7%；
- 日文罗马字的 code point 是混写的 2.48 倍，但 UTF-8 字节反而更少，物理宽度只多约 13.4%；
- 不更换语言，仅更换文字，就足以改变“紧凑度”排名。

这类转写并非完全无损。汉语拼音不能唯一恢复所有同音汉字；日文全假名和罗马字会丢失汉字提供的词义与切分线索。它们适合隔离表面编码成本，但不能假定认知解码条件完全相同。

### 4.7 Unicode 中至少有八种容易被统称为“字符”的对象 {.no-toc}

| 对象 | 定义 | 与其他层级的关系 |
|---|---|---|
| 语言学单位 | 音位、拍、音节、语素、词 | 不由 Unicode 定义 |
| 正字法基本单位 | 字母、假名、汉字等 | 可对应一个或多个语言学单位 |
| Unicode 抽象字符 | 标准所编码的抽象文本成分 | 不等于具体字形 |
| code point | Unicode 代码空间中的整数位置 | 一个用户感知字符可由多个 code point 组成[54] |
| code unit | 某编码形式的最小存储单位 | UTF-8 为 8 bit，UTF-16 为 16 bit |
| byte | 8 bit 存储单位 | 一个 Unicode 标量值在 UTF-8 中用 1—4 byte |
| 扩展字素簇 | 对用户感知“一个字符”的可编程近似 | 常用于光标移动、删除和字符限制[56] |
| glyph/字形 | 字体实际绘制的形状 | 一个字形可对应一个或多个字符，反之亦可[54] |
| 显示列/行进宽度 | 终端列或字体排版推进距离 | 受 East Asian Width、字体和布局控制[55] |

Unicode 明确区分字符和字形：code point 是抽象字符的编码位置；glyph 是屏幕或纸面上的渲染形状；两者不存在普遍的一一对应。[54] 所以“中文一个字符占一个格，英文一个字母占一个格”既不是 Unicode 定义，也不是比例字体排版的事实。

### 4.8 code point、字素簇和 UTF-8 byte 会因规范化而分离 {.no-toc}

| 可见形式 | code point 序列 | code point 数 | 扩展字素簇 | UTF-8 byte |
|---|---|---:|---:|---:|
| `が` | U+304C | 1 | 1 | 3 |
| `が` | U+304B U+3099 | 2 | 1 | 6 |
| `é` | U+00E9 | 1 | 1 | 2 |
| `é` | U+0065 U+0301 | 2 | 1 | 3 |
| `👨‍👩‍👧‍👦` | 7 个 code point，含 3 个 ZWJ | 7 | 1 | 25 |

前两行和中间两行分别是规范等价表示：用户通常看到一个字符，但底层 code point 与 byte 数不同。Unicode 文本分割标准建议用扩展字素簇近似用户感知字符；规范化标准定义 NFC、NFD、NFKC 和 NFKD。[56][57]

可复算比较应至少：

1. 声明输入是否规范化；
2. 通常先用 NFC 比较 code point 和 byte；
3. 同时报告扩展字素簇；
4. 不要盲目用 NFKC，因为兼容规范化可能消除全角/半角、数学符号等有意义的区别。

### 4.9 词熵研究与 CJK 切分问题 {.no-toc}

Bentz 等（2017）分析三个平行语料库，共约 4.5 亿词、1,916 个文本、1,259 种语言，研究词级熵、可学习性与表达力。其样本中没有语言的词元一阶熵低于约 6 bit/word；加入前文条件后，平均不确定性下降约 3 bit/word。[10]

但该研究把“词”操作化为由空格等边界分隔的字母数字 UTF-8 字符串。该定义对英文等空格分词语言较自然，对中文和通常不以空格分词的日文则不等价。分词至少可以选择：

1. 原始 code point 或字素簇；
2. 语言学词；
3. 语素；
4. 音节或拍；
5. 数据驱动子词。

五种粒度可能给出五种排名。严谨研究应报告切分敏感性，而不是把空格当作跨语言的天然词界。

### 4.10 平行语料的证据价值与局限 {.no-toc}

#### 联合国平行语料库 {.no-toc}

UN Parallel Corpus 收录 1990—2014 年联合国文件的六种官方语言版本，并提供句级对齐：阿拉伯语、中文、英语、法语、俄语和西班牙语。[11][49] 优势是内容平行、编辑质量高、法律与政策文本规模大；局限是日语不在六种官方语言中、文体高度正式、翻译不必逐词等长，并会出现源语言和机构规范造成的“翻译语”特征。[14][43]

#### JParaCrawl {.no-toc}

JParaCrawl v3.0 报告超过 2,100 万个独特英—日平行句对。[12] 它补足联合国语料的日语缺口，但网页抓取和自动对齐会引入模板、重复、错配和领域偏差。

#### OpenSubtitles {.no-toc}

OpenSubtitles2018 覆盖约 370 万份字幕和 60 种语言，可近似研究对话体。[13] 但字幕受时长、断行、删减和翻译改写约束，不是未经编辑的自然口语转录。

平行语料只能控制“近似相同内容”，不能自动控制字体、排版、翻译方向、语用信息和语言学单位切分。

### 4.11 联合国《世界人权宣言》第一条：code point 与 UTF-8 byte {.no-toc}

以下英、中、日文本取自 OHCHR 的相应页面；西班牙文加入用于检验 Reddit 对英—西长度的判断。[40][41][42][62] 英文、中文和西班牙文是联合国官方语言版本；日文是 OHCHR 收录的翻译，但不属于联合国六种官方语言，OHCHR 也只对六种官方语言版本承担准确性保证。[43] 日文网页中由旧版排版换行产生的词内空格在计数前被删除。

> **英文**<br>
> All human beings are born free and equal in dignity and rights. They are endowed with reason and conscience and should act towards one another in a spirit of brotherhood.

> **西班牙文**<br>
> Todos los seres humanos nacen libres e iguales en dignidad y derechos y, dotados como están de razón y conciencia, deben comportarse fraternalmente los unos con los otros.

> **中文**<br>
> 人人生而自由,在尊严和权利上一律平等。他们赋有理性和良心,并应以兄弟关系的精神相对待。

> **日文**<br>
> すべての人間は、生まれながらにして自由であり、かつ、尊厳と権利とについて平等である。人間は、理性と良心とを授けられており、互いに同胞の精神をもって行動しなければならない。

计数规则：NFC；原始字符串按 Unicode code point；“去空格标点”删除 Unicode 空白和 `P*` 标点类别；字节采用 UTF-8；不做语言学分词。

| 语言 | 原始 code point | 去空格标点 code point | 原始 UTF-8 byte | 去空格标点 UTF-8 byte | 空格切分 token |
|---|---:|---:|---:|---:|---:|
| 英文 | 170 | 139 | 170 | 139 | 30 |
| 西班牙文 | 171 | 141 | 173 | 143 | 28 |
| 中文 | 43 | 39 | 125 | 117 | 1 |
| 日文 | 85 | 78 | 255 | 234 | 1 |

由此可见：

- 按去标点 code point，中文为英文的 28.1%，日文为 56.1%，西班牙文为 101.4%；
- 按去标点 UTF-8 byte，中文上升到英文的 84.2%，日文为 168.3%，西班牙文为 102.9%；
- 英文和西班牙文在这条官方平行文本中几乎等长，西班牙文空格 token 反而少两个；
- 中日文被空格法算成一个 token，说明该“词数”指标失效。

这不是语言总体平均值，只是一个可复算的反例：它足以否定“西班牙语翻译总是远长于英语”这种绝对说法，却不足以证明两种语言在所有体裁中等长。

### 4.12 同一联合国文本：code point 优势不等于页面优势 {.no-toc}

再对同一组原始文本计算常见终端列宽和字体行进宽度：

| 语言 | code point | UTF-8 byte | 终端列 | Noto Sans CJK 行进宽度（em） |
|---|---:|---:|---:|---:|
| 英文 | 170 | 170 | 170 | 79.87 |
| 西班牙文 | 171 | 173 | 171 | 81.05 |
| 中文 | 43 | 125 | 84 | 41.56 |
| 日文 | 85 | 255 | 170 | 84.58 |

解释：

- 日文 code point 只有英文的 50%，但多数日文字符按终端惯例占两列，因此总列数与英文相同；在该比例字体中，日文行进宽度比英文多约 5.9%；
- 中文 code point 只有英文的 25.3%，但行进宽度约为英文的 52.0%。中文仍明显紧凑，只是没有“字符数比值”看起来那么极端；
- 西班牙文和英文的 code point 与实际宽度均接近；
- “一个假名等于两个英文字母，所以每页更短”只在 token 数层面有时成立，在物理行宽层面并不成立。

测量条件：Noto Sans CJK Regular 2.004，100 px；字体文件 SHA-256 为 `b76b0433203017ca80401b2ee0dd69350349871c4b19d504c34dbdd80541690a`。行进宽度是排版推进距离，不是墨迹包围盒，也未模拟换行、行高、字距、竖排或真实纸张。

Unicode East Asian Width 是为东亚旧编码互操作和终端宽度提供的属性，不是精确像素宽度；全角/半角也不是所有 Unicode 字符固有的物理属性。[55] 此外，临床可辨识研究表明，复杂汉字和拉丁字母视力表不能假定在相同角尺寸下具有同等可辨识性；Wang 等对 1,243 个繁体字和 42 只健康眼的研究发现，汉字可辨识字号需要相对于 ETDRS 字母视力阈值作显著放大。[58] 该研究不是正文阅读速度实验，不能直接给出“每页密度比例”，但足以说明字符数不能替代可读版面测试。

### 4.13 英语—西班牙语话语长度：不是“自由语序”问题 {.no-toc}

Reddit 举出的 `The missile avoidance generator expenses` 本身就有结构和语义问题。它至少可能被理解为不同的嵌套关系，而名词之间的关系没有被显式标明。Nakov 的综述把类似长名词复合视为英语文本自动解释中的主要难题，因为需要恢复内部句法和语义关系。[59]

英语名词串的较短表面形式来自：

- 名词—名词复合高度生产；
- 名词可置于另一名词前作定语；
- `of`、`for`、`used for`、`located in` 等关系经常留给语境推断。

这与英语“语序自由”相反。现代英语的基本 SV(O) 语序相对固定；日语等语言在成分顺序上反而允许更多变体。[60]

西班牙语名词短语更常用 `de` 等介词和名词后修饰把关系显式化；西班牙语也有复合构词，但其生产性和形式分布不同于英语。[61] 这只说明**某类名词短语**可能更长，不说明所有西班牙语句子更长。西班牙语动词屈折编码人称和数，并广泛允许省略可恢复的主语；英语通常要求显式主语。[63] 长度成本在不同结构中重新分配。

中文和日文也能形成紧凑名词串，例如 `导弹防御系统开发成本` 和 `ミサイル防衛システム開発費`。因此，名词复合不是英语独占能力。比较必须先固定精确语义，再让多名母语者给出自然表达，并报告歧义和理解，而不能从一条机器翻译或一个生造短语推出语言规则。

### 4.14 面向“每页紧凑度”的最低测量规范 {.no-toc}

真正比较每页或每屏的信息密度，应同时固定或报告：

1. 平行内容及翻译方向；
2. Unicode 规范化形式；
3. code point、扩展字素簇和语言学词/语素；
4. 字体名称、版本、语言区域字形、字号、字重和字距；
5. 版心宽高、行距、段距、标点挤压、断行和书写方向；
6. UTF-8/UTF-16 字节及压缩后大小；
7. 阅读速度、理解正确率和最小可辨识字号。

只报告“每页字符数”最多是排版描述，不能单独作为语言信息密度或认知效率证据。

### 4.15 可见紧凑度、编码紧凑度和语义紧凑度仍会分离 {.no-toc}

以“人工智能”为例：

| 表达 | 可见字符（含空格） | UTF-8 byte | 说明 |
|---|---:|---:|---|
| `artificial intelligence` | 23 | 23 | 两个英文词，ASCII |
| `人工智能` | 4 | 12 | 四个汉字 |
| `人工知能` | 4 | 12 | 四个日文汉字 |
| `AI` | 2 | 2 | 三种语言环境都可使用的缩写 |

共同体一旦采用 `AI`，三种语言的字符和字节差异几乎消失。密度因此也是术语成熟度和共享知识的函数，而不只是语法或文字体系属性。


## 5. 对话、写作与体裁

### 5.1 口语不是低质量写作，写作也不是压缩口语 {.no-toc}

Chafe 与 Tannen 的综述、Biber 的多维语体分析均表明，口语—书面语不是单一轴上的简单对立。[15][16] 不同体裁在下列维度上交叉变化：

- 互动性与信息性；
- 情境依赖与脱离情境；
- 即时生成与可编辑；
- 明确指称与共同知识；
- 名词化、从句复杂度和词汇多样性；
- 立场、礼貌和人际功能。

一般而言，编辑后的学术或行政写作常有较高词汇密度和名词化；自然对话常有更多代词、语气词、重复、修正和省略。[17] 但新闻播报、即时通信、小说对白和法律口头陈述会跨越这种二分。

### 5.2 对话中的“冗余”常是纠错码 {.no-toc}

对话中的重复、应答、重述和确认类似于通信系统中的冗余：

- “你说周四，不是周五，对吗？”
- “对，周四下午。”

第二轮没有引入很多新命题，却显著降低了错误概率。若只计算新词或新命题，它显得低密度；若计算成功协调，它可能提高任务效率。跨语言比较若忽略修复机制和轮替，会把可靠性误判为浪费。[47]

### 5.3 同一命题的显式成分不同 {.no-toc}

在已知语境下，“我已经把它发给他了”可有下列表达：

| 语言 | 较完整形式 | 语境充分时的省略形式 | 典型显式信息 |
|---|---|---|---|
| 中文 | 我已经发给他了。 | 已经发了。 | 体/状态粒子、语序；论元可省 |
| 日文 | 私はもう彼にそれを送った。 | もう送った。 | 助词、动词屈折；论元常可省 |
| 英文 | I have already sent it to him. | Sent it already. | 标准完整句通常显式主语、代词与完成结构；口语片段也可省略 |

这不是长度比赛。三种语言都能根据语境在完整形式与片段之间移动。区别在于何种省略被视为中性、何种语法范畴通常显式，以及接收者可依赖多少共同知识。

### 5.4 “相同内容”必须说明是否包含语用信息 {.no-toc}

比较下列请求：

- 英文：`Could you send it?`
- 中文：`能发一下吗？`
- 日文：`送っていただけますか。`

若目标内容只定义为“请求发送”，三句大致对应；若目标还包括说话人—听话人的社会关系、礼貌等级、负担缓和和语气，三句并非完全等价。日语较长的礼貌形式不一定是冗余，而可能编码了研究者在“命题内容”定义中删掉的社会信息。


## 6. 认知科学证据

### 6.1 眼动与知觉广度 {.no-toc}

经典眼动研究估计，英文阅读的有效知觉广度大致不对称地覆盖注视点左侧约 3—4 个字符、右侧约 14—15 个字符；中文阅读研究得到约左侧 1 个汉字、右侧 3 个汉字的范围。[18][19]

不能由此得出“英文大脑一次处理更多信息”或“中文每字信息恰好是英文四倍”。原因包括：

- 英文字符比汉字更小，多个字母组成一个词；
- 中英文视觉复杂度和词边界线索不同；
- 实验中的字体大小、显示角度和掩蔽方法会影响范围；
- 知觉广度测量的是眼动中可用的预视信息，不是语义 bit 数。

更稳妥的解释是：阅读系统会根据文字单位和词切分结构调整采样窗口。

### 6.2 日文混合文字的切分优势 {.no-toc}

Sainio 等（2007）比较有空格与无空格的日文阅读，发现空格明显帮助纯平假名文本的词识别和眼动引导，但对常规汉字—平假名混写文本的帮助较小。[20] 这说明汉字与假名的视觉类别转换本身提供了词边界线索。

同一个日语词若全部写成假名，字符数可能增加，视觉切分也会改变。例如：

- `情報密度`
- `じょうほうみつど`

两者属于同一语言、近似同一读音和意义，但可见字符数、字符类别和识别线索不同。所谓“日语字符密度”不能脱离具体书写形式。

Shafiullah 与 Monsell（1999）在日语词命名任务中发现，汉字与假名脚本切换会造成约 13 毫秒的平均切换减速，并伴随准确率下降。[21] 这类结果说明混合文字带来切分资源，也带来脚本转换成本；不能只计算字数。

### 6.3 发音时长、工作记忆与数字广度 {.no-toc}

工作记忆研究长期发现，短时序列回忆受发音时长和默读复述速度影响，即“词长效应”。[23] 汉语数字名称通常较短，中文母语者在某些数字广度任务中可表现出优势；跨语言实验也发现母语项目的复述速度与记忆广度相关。[22]

但这不等于“中文提高一般智力”：

- 优势高度依赖材料是否能用本语言快速复述；
- 视觉编码、熟悉度、教育和策略会参与；
- 换成不可命名图形、空间任务或不同词长材料，排序可能改变；
- 后续研究认为单纯用发音时长不能解释全部中英文差异。

因此，这是**任务—编码匹配效应**，不是语言整体的认知优越性。

### 6.4 阅读脑网络：共享机制与文字适应 {.no-toc}

跨文字系统研究通常发现，阅读依赖共同的语言和视觉网络，同时对正字法深度、视觉复杂度和形—音映射进行系统特化。[26] 中文阅读更依赖字形—语素和音节映射；英文更依赖字母串中的音位与词形模式；日文在汉字和假名之间调用部分不同但重叠的加工路径。

Tan 等（2005）发现，汉字书写技能和正字法意识与中文阅读技能有重要关系。[27] 该结果说明训练方式会塑造阅读网络，不能被转写为“汉字天然提升抽象思维”。

### 6.5 命题密度不是语言智力 {.no-toc}

“idea density”通常指单位词数中的命题或观念数量。Nun Study 发现，93 名修女年轻时自传中的观念密度和语法复杂度与晚年认知表现相关；病理子样本只有 14 名去世受试者，因果解释受样本和选择机制限制。[24]

CPIDR 等自动工具能用词性和句法规则估计英文命题密度。[25] 但将其跨语言使用会遇到：

- 中文和日文的词切分不同；
- 零论元和名词化分析不同；
- 同一命题可以被编码为复合词、从句或语境推断；
- 自动分析器精度和训练资源不同。

“某文本命题密度高”是文本特征，不是其语言或作者的一般智力等级。


## 7. 信息论、编码学与计算语言学

### 7.1 源编码与信道编码的类比 {.no-toc}

自然语言同时承担两类功能：

- **源编码**：用较短、较可预测的形式表达常见意义；
- **可靠传输**：通过冗余、重复、语法标记和修复抵抗噪声。

Zipf 定律和后续研究显示，高频或在语境中更可预测的词往往更短；词长更好地受信息内容而非单纯频率解释。[9][44] 这反映长期使用压力下的效率，但自然语言不会像无噪声压缩文件那样消除全部冗余，因为人类通信存在听错、注意波动、共同知识不一致和社会目标。[45]

### 7.2 UTF-8：汉字紧凑不等于字节紧凑 {.no-toc}

UTF-8 对 U+0000—U+007F 使用一个字节，对多数常用中日韩统一表意文字所在的基本多文种平面代码点通常使用三个字节。[36][48]

因此：

- ASCII 英文字母常为 1 byte；
- 常用汉字、平假名和片假名通常为 3 bytes；
- 同样的视觉字符数不等于同样的存储量；
- Unicode 规范化、组合字符、标点和换行还会继续影响字节数。

联合国第一条示例已经显示：中文按 code point 比英文短很多，但按 UTF-8 字节只短约 15.8%；日文按字节反而更长。

### 7.3 压缩率不是纯语言熵 {.no-toc}

用 gzip、Brotli 或神经压缩器比较语言时，结果同时受以下因素影响：

- 输入编码和规范化；
- 语料长度；
- 重复模板；
- 压缩器窗口和字典；
- 字符集大小；
- 文体和主题；
- 是否混合数字、代码与专名。

压缩后字节数可作为工程指标，但不能直接解释为“人类实际感知的信息量”。比较时至少应同时报告原始 code point、UTF-8 byte、压缩 byte 和模型估计的条件熵。

### 7.4 神经语言模型 token 是模型变量 {.no-toc}

Petrov 等（2023）比较多种分词器，发现同一内容的翻译在极端语言对之间可出现最高约 15 倍的 token 长度差；即便字符级或字节级模型，某些语言对的编码长度也可相差四倍以上。[37] Ahia 等（2023）在 22 种语言上进一步展示，token 数差异会传导到 API 成本、上下文容量和效用。[38]

这些结果的正确解释是：

- 分词器训练语料不均衡；
- 词表容量在语言之间分配不均；
- 常见字符串是否被合并成一个子词取决于模型；
- 语言的书写系统与分词算法发生交互。

Rust 等（2021）在九种语言和五类任务中发现，将多语模型原有分词器替换为专门的单语分词器，几乎所有语言和任务的表现都得到改善。[39] 因此，`token/句` 不应被称为某语言的内在信息密度。

### 7.5 计算系统中的三种公平性 {.no-toc}

| 工程问题 | 不公平来源 | 应报告的指标 |
|---|---|---|
| 计费 | 同内容 token 数不同 | token、字符、字节、语义任务量 |
| 上下文窗口 | 某语言更快耗尽 token | 可容纳句数/命题数，而非只报 token |
| 推理延迟 | 输出被过度切碎 | 每语义单位 token、每秒生成字符/音节 |
| 模型质量 | 某语言词表覆盖低 | fertility、未知/稀有子词率、下游准确率 |

工程上应修正 tokenizer，而不是从不公平 tokenizer 推断语言劣势。


## 8. 同一内容在三种语言中的异同

### 8.1 “相同内容”有三个层次 {.no-toc}

1. **命题核心相同**：谁对谁做了什么；
2. **语法信息相同**：时态、数、定指、格、体等都对齐；
3. **完整交际效果相同**：礼貌、立场、信息结构、含蓄程度和社会关系也对齐。

多数平行语料只能近似保证第 1 层；法律翻译努力接近第 2 层；第 3 层通常需要改写，不可能靠逐词等长实现。

### 8.2 强制表达范畴不同会改变长度 {.no-toc}

| 范畴 | 中文常见方式 | 日文常见方式 | 英文常见方式 | 长度影响 |
|---|---|---|---|---|
| 主语/宾语 | 语境明确时可省 | 语境明确时常省 | 标准完整句多显式主语；口语片段可省 | 对话中中日常更短，但并非绝对 |
| 时体 | 体词、时间词、语境 | 动词屈折与助动成分 | 动词屈折、助动词、时间词 | 同一时间关系可分散在不同单位 |
| 定指/冠词 | 通常无冠词系统 | 通常无冠词系统 | `a/the` 常需选择 | 英文常增加功能词，同时提供定指线索 |
| 格/语法关系 | 语序、介词/处置结构 | 助词显式标记 | 语序、介词、代词格残余 | 日语助词增加表面单位但降低关系歧义 |
| 数量分类 | 数词常配量词 | 数词常配助数词 | 可数名词数与冠词更显式 | 三者都可能在不同位置增加标记 |
| 礼貌/身份 | 词汇、称呼、句式 | 词汇、敬语形态、句尾形式较系统 | 情态、称呼、间接策略 | 若社会信息纳入内容，日语形式不应算冗余 |

### 8.3 翻译长度不只由语言决定 {.no-toc}

同一原文的长度还受以下因素控制：

- 源语言方向；
- 直译或归化策略；
- 法律等值或通俗解释目标；
- 专名是否音译；
- 术语是否已词汇化；
- 标点和数字规范；
- 受众知识；
- 是否保留重复与修辞。

可靠比较应使用双向翻译：既比较英→中/日，也比较中/日→英，并加入各语言原创文本，否则源语言结构可能系统性偏置结果。[14]


## 9. 同一语言内部的密度变化与可拓展性

### 9.1 体裁差异可以大于语言差异 {.no-toc}

| 内容类型 | 统计熵 | 命题密度 | 解码成本 | 典型原因 |
|---|---|---|---|---|
| 法律模板 | 局部条件熵较低 | 可高 | 高 | 术语密集、句法嵌套、重复保障精确性 |
| 日常对话 | 词汇密度较低 | 中低 | 对参与者低、旁观者高 | 共享语境、指示、省略、修复 |
| 科学摘要 | 高 | 高 | 中高 | 名词化、术语、压缩背景 |
| 教材 | 中 | 中 | 较低 | 有意重复、定义和例示 |
| 诗歌/格言 | 可高 | 表面可低 | 高 | 歧义、隐喻、文化知识 |
| 程序代码/公式 | 符号密度高 | 对专家高 | 对非专家极高 | 形式语义与专业约定 |

同一语言中的题材、作者、受众和编辑程度，可能造成比跨语言平均值更大的差异。2011 与 2019 的实验也都显示文本和说话人是重要变异来源。[3][4]

### 9.2 中文并非无限“旧字组合即可” {.no-toc}

中文复合词的透明度有梯度：

- 高透明：`量子计算`、`基因编辑`；
- 部分透明：`手机`、`软件`；
- 词汇化或历史化：`东西`、`马虎`；
- 音译/借词：`咖啡`、`沙发`；
- 字母词：`DNA`、`GDP`、`AI`。

组合能力不能保证读者立刻理解。`神经网络` 对机器学习共同体透明，对无相关背景者仍需定义。概念扩展能力来自社会共同体建立约定，不只来自字面可分析性。

### 9.3 英文也不必为每个概念发明原子词 {.no-toc}

`artificial intelligence`、`machine learning`、`large language model`、`gene drive` 都由既有词组合。英文还可无限递归构造名词短语，例如 `low-resource multilingual speech recognition system`。其表面字符较长，部分原因是字母拼写和空格分词，而不是概念无法组合。

### 9.4 日文在借词、汉语词与本土形态间切换 {.no-toc}

日语可将同一领域概念表达为：

- 汉字复合：`人工知能`；
- 片假名借词：`ニューラルネットワーク`；
- 混合：`AI技術`；
- 缩略：`生成AI`。

借词表面较长不代表概念系统弱。片假名还可以标示术语来源、现代性或领域身份，这些也是语用信息。

### 9.5 缩写使语言间差异动态变化 {.no-toc}

一个新概念初期可能使用长解释，稳定后形成术语，再进一步缩写。密度随知识扩散而上升：

\[
\text{长定义} \rightarrow \text{固定术语} \rightarrow \text{缩写/符号}
\]

这说明“密度”不仅是语法属性，也是共同知识积累的结果。专业社群中的高密度文本对外部读者可能完全不可解码。


## 10. 主要反驳与平衡

### 10.1 “中文一个字就是一个概念，所以信息密度最高” {.no-toc}

**部分成立：** 汉字常提供语素级信息，中文正式文本按可见字符往往短。<br>
**不成立之处：** 字与词、意义、音节不是严格一一对应；多字词大量存在；统计信息需要概率模型；UTF-8 字节和口语时间可给出不同排名。

### 10.2 “日语最冗长，因此效率最低” {.no-toc}

**部分成立：** 在 2011 和 2019 控制材料中，日语每音节信息较低，表达同内容通常需要更多音节。<br>
**不成立之处：** 日语音节速率最高，bit/s 接近普通话；音节并非日语最自然的唯一单位；助词、屈折和敬语编码了语法与社会信息。

### 10.3 “英语必须持续造新词，而中文只需组合旧字” {.no-toc}

**错误。** 三种语言均大量复合、派生、借词、仿译和缩略。真正差异是：

- 一个复合表达按正字法写成一个词还是多个词；
- 倾向意译还是音译；
- 术语是否使用汉字、片假名或拉丁字母；
- 社群何时把组合词词汇化。

### 10.4 “约 39 bit/s 证明所有语言完全等效” {.no-toc}

**过度推论。** 2019 研究显示的是受控朗读条件下的统计收敛：平均约 39.15 bit/s，标准差仍约 5.10；语言、说话人和文本均有差异。自然对话、歌唱、演讲、失语、噪声环境和第二语言使用不在其结论范围内。[4]

### 10.5 “大模型 token 更少的语言更高级” {.no-toc}

**错误。** token 是词表和训练数据的产物。同一语言换一个分词器，token 数和性能都会变化。[37][38][39]

### 10.6 “字符更少就更易读” {.no-toc}

**错误。** 可读性还受词频、句法嵌套、背景知识、歧义、字体、词边界、眼动规划和工作记忆影响。压缩可能提高版面效率，同时提高解码负荷。

### 10.7 “字符库存越大，每个符号的信息就必然越多” {.no-toc}

**错误。** 库存规模只给出等概率独立编码的理论上限。自然文本中的实际每符号熵取决于频率分布、上下文条件、切分粒度和语料；大量罕见字符不会平均提升所有常用字符的信息量。更大的库存还增加学习和识别成本。

### 10.8 “code point 更少就等于页面更短” {.no-toc}

**错误。** 日文《世界人权宣言》第一条的 code point 只有英文一半，但在同一 Noto Sans CJK 字体下行进宽度略大，按常见终端规则两者列数相同。页面比较必须测量字体和版面，而不能只数 code point。

### 10.9 “UTF-8 字节更多说明语言更冗长” {.no-toc}

**错误。** UTF-8 为 ASCII 提供 1 byte 表示，多数基本平面汉字和假名使用 3 byte；这是编码格式的工程选择。换成 UTF-16、压缩字节或模型 token，比例会再次变化。

### 10.10 “认知实验中的数字广度差异等于一般智力差异” {.no-toc}

**错误。** 数字广度受数字名称长度、复述速度、任务策略和教育经验影响。它测量特定的短时言语编码任务，不是语言或族群的总体智力。


## 11. 现实影响

### 11.1 翻译与本地化 {.no-toc}

- UI 设计不能用统一字符数上限替代实际版面测试；中文视觉较短，德语等复合词或日文假名/片假名术语可能更长。
- 字幕应同时考虑字符、阅读速度、镜头时长和断行，而非只按词数。
- 法律翻译必须优先功能等值和歧义控制，不能为了“密度”删除重复。
- 机器翻译评测应记录翻译方向和源语言，避免把翻译腔当作目标语言属性。

### 11.2 语音识别、合成与同声传译 {.no-toc}

- 语音系统应区分音节、拍、音素和词速；同一个统一“words per minute”指标对中、日、英并不公平。
- 同声传译要管理信息峰值，而非只追求平均语速。
- 对日语合成，拍长和句尾形式是自然度的重要变量；对普通话，声调错误会改变词汇辨识；对英语，重音弱化和连读影响可懂度。

### 11.3 大语言模型与计算成本 {.no-toc}

- 应报告每个语义任务的 token 成本，而非只报每千 token 单价；
- 多语模型应审计 tokenizer fertility、上下文有效容量和生成延迟；
- 改善词表分配、采用公平的子词学习或字节/字符混合架构，通常比把差异归因于语言更合理；
- 评测应同时提供字符、字节、token 和任务完成率。

### 11.4 教育与认知测试 {.no-toc}

- 跨语言记忆测验必须控制项目的发音时长、频率和视觉复杂度；
- 阅读速度不应只按“字/分钟”或“词/分钟”横向比较；
- 汉字、假名和字母教学需要不同的正字法意识训练，但不应被排列为认知高低等级。

### 11.5 数据压缩与数据库设计 {.no-toc}

- 用户可见长度限制应优先按扩展字素簇处理，并另行校验编码字节；
- 数据库字段应按字节和字素簇设计，不能假定一个字符等于一个字节；
- 文本压缩实验需要统一 Unicode 规范化和语料体裁；
- 搜索和索引系统必须明确中文/日文分词方案，否则“词频”和“词熵”不可复现。


## 12. 建议的统一比较协议

要获得比“数一数字符”更可靠的中—日—英比较，可采用下列协议。

### 12.1 语料设计 {.no-toc}

1. 每种语言同时包含原创文本与双向翻译文本；
2. 至少覆盖自然对话、字幕、新闻、学术、法律和技术说明；
3. 同一内容由多名译者独立翻译；
4. 记录源语言、译者、地区变体、文体和受众；
5. 口语同时采集朗读与自发对话。

### 12.2 多粒度计量 {.no-toc}

每个样本至少报告：

- 扩展字素簇、code point、UTF-8/UTF-16 byte；
- 字体行进宽度、实际换行后行数、版面面积与最小可辨识字号；
- 字符、词典词、语素、音节、日语拍；
- 一阶熵、条件熵、现代语言模型 surprisal；
- 发音总时长、有效发声时长、停顿时长；
- 命题数、指称实体数、语法特征数；
- 多种 tokenizer 下的 token 数；
- 压缩后字节数。

### 12.3 行为结果 {.no-toc}

密度必须与以下结果联合：

- 理解正确率；
- 反应时；
- 眼动或回视；
- 记忆保持；
- 噪声下恢复率；
- 对话任务成功率；
- 主观努力度。

### 12.4 统计模型 {.no-toc}

使用分层模型，将下列因素设为随机或固定效应：

- 语言；
- 说话人/作者；
- 文本/命题；
- 翻译者；
- 体裁；
- 源语言方向；
- 熟练度；
- 分词和熵估计方法。

最终结论应是一个**敏感性区间**，而不是单一排行榜。例如：

> 在法律平行文本、按无标点 code point 计，中文更短；在 UTF-8 byte 计，差距显著缩小；在控制朗读、按 bit/s 计，三语差异收敛；在某一模型 token 计，排名随 tokenizer 改变。

这类条件化结论才具有可重复性。


## 13. 综合判断

### 13.1 可以较有把握地成立的命题 {.no-toc}

1. **每音节信息与音节速率存在跨语言补偿关系。** 日语通常位于“低单位信息—高速度”一端，普通话位于“高单位信息—低速度”一端，英语居中或偏高。[3][4]
2. **每秒信息率比每音节信息更接近。** 2019 年 17 语言研究支持约 39 bit/s 附近的收敛，但不是精确相等。[4]
3. **中文在可见字符上经常最紧凑，但 code point 比例会夸大页面比例。** 这与汉字的语素级功能、无空格书写、低屈折和复合构词有关；字体宽度、字号与可辨识性仍需另测。
4. **日语“一个假名等于两个英文字母”不是稳定换算。** 假名主要对应拍，标准日文是汉字—假名混写；同一日语改成全假名或罗马字即可显著改变字符、字节和版面长度。
5. **书写紧凑性会在字节、token 和认知成本上被重估。** 联合国第一条示例和 tokenizer 研究都显示单位切换可改变幅度或排名。[37][40][41][42]
6. **三种语言都具有开放的概念扩展能力。** 复合、派生、借词、仿译、缩略与语境重用是人类语言的共同资源。[28][31][33]
7. **语言没有可独立于任务定义的“智能密度”。** 可比较的是文本、说话方式、编码方案或任务中的效率，而不是语言或使用者的智力等级。

### 13.2 不能由现有证据支持的命题 {.no-toc}

- 中文在所有意义上信息密度最高；
- 日语总体传输效率最低；
- 英文不能组合旧词表达新概念；
- 所有人类语言具有严格相同的 bit/s；
- 汉字导致更高智力或英文导致更强逻辑；
- 大模型 token 数可以测量语言本体优劣；
- 单条联合国文件足以估计整种语言的平均密度。

### 13.3 最终结论 {.no-toc}

中、日、英三语没有一个稳定的总排序。更准确的关系是一个多维权衡：

\[
\text{视觉紧凑性} \neq \text{单位音节信息} \neq \text{每秒信息率} \neq \text{语义命题密度} \neq \text{理解效率} \neq \text{计算编码成本}
\]

按 code point，同内容的现代中文通常最短，日文常居中，英文常较长；但 code point 不是页面面积。本文的联合国样例中，日文 code point 只有英文一半，字体行进宽度却略大；中文的 code point 约为英文四分之一，行进宽度约为一半。按 UTF-8 字节，差距再次改变；按口语音节，普通话和英语比日语更密；按每秒统计信息，日语通过速度补偿，三者趋近；按模型 token，结果主要是分词器设计。任何脱离单位、语料、文字、字体、语境、受众和任务的“语言密度排行榜”都缺乏科学意义。

若必须使用“智能密度”一词，唯一可辩护的定义是：**在给定任务中，以可接受的时间、错误率和认知负荷，成功恢复目标意义的效率。** 在这个定义下，优势属于具体表达与具体场景，而不属于某一种语言整体。

```python
import unicodedata


def count_text(text: str) -> dict[str, int]:
    stripped = "".join(
        ch for ch in text
        if not ch.isspace()
        and not unicodedata.category(ch).startswith("P")
    )
    return {
        "codepoints_raw": len(text),
        "codepoints_no_space_punct": len(stripped),
        "utf8_bytes_raw": len(text.encode("utf-8")),
        "utf8_bytes_no_space_punct": len(stripped.encode("utf-8")),
        "whitespace_tokens": len(text.split()),
    }
```

## 附录 B：Coupé 等（2019）数据复算

使用论文补充材料附带的制表符分隔文件 `InfoRateData.csv`（数据入口见 <https://doi.org/10.1126/sciadv.aaw2594>）：

- 本次分析文件 SHA-256：`f4dde6def980ed4c461ad8fd666af64cbbaa967a2ac73702855fc70484c0732a`
- 计算：`SR = NS / Duration`
- 计算：`IR = SR * ID`
- 汇总：对各语言每条录音等权取算术平均

```python
import pandas as pd


df = pd.read_csv("InfoRateData.csv", sep="\t")
df["SR"] = df["NS"] / df["Duration"]
df["IR"] = df["SR"] * df["ID"]

result = (
    df[df["Language"].isin(["CMN", "ENG", "JPN"])]
    .groupby("Language")
    .agg(
        recordings=("Language", "size"),
        speakers=("Speaker", "nunique"),
        ShE=("ShE", "mean"),
        ID=("ID", "mean"),
        SR=("SR", "mean"),
        IR=("IR", "mean"),
        IR_SD=("IR", "std"),
    )
)
print(result)
```

该复算不是对论文混合效应分析的再拟合，也未处理语言内重复测量权重；它只提供可审计的描述性统计。

## 附录 D：Unicode、终端列与字体宽度复算

以下代码演示本文新增表格的计量方法。依赖：Python 3、`regex`、`wcwidth`、Pillow。`regex` 的 `\X` 用于扩展字素簇；`wcwidth` 估计终端列；Pillow 的 `getlength` 测量字体行进宽度。

```python
from __future__ import annotations

import hashlib
import unicodedata
from pathlib import Path

import regex
from PIL import ImageFont
from wcwidth import wcswidth


def strip_space_punctuation(text: str) -> str:
    return "".join(
        ch
        for ch in text
        if not ch.isspace()
        and not unicodedata.category(ch).startswith("P")
    )


def measure(text: str, font: ImageFont.FreeTypeFont) -> dict[str, float | int]:
    text = unicodedata.normalize("NFC", text)
    stripped = strip_space_punctuation(text)
    return {
        "codepoints": len(text),
        "grapheme_clusters": len(regex.findall(r"\X", text)),
        "utf8_bytes": len(text.encode("utf-8")),
        "terminal_columns": wcswidth(text),
        "advance_em": font.getlength(text) / font.size,
        "stripped_codepoints": len(stripped),
        "stripped_utf8_bytes": len(stripped.encode("utf-8")),
        "whitespace_tokens": len(text.split()),
    }


font_path = Path("/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc")
font = ImageFont.truetype(str(font_path), size=100, index=2)
print(hashlib.sha256(font_path.read_bytes()).hexdigest())

texts = {
    "English": (
        "All human beings are born free and equal in dignity and rights. "
        "They are endowed with reason and conscience and should act towards "
        "one another in a spirit of brotherhood."
    ),
    "Spanish": (
        "Todos los seres humanos nacen libres e iguales en dignidad y derechos y, "
        "dotados como están de razón y conciencia, deben comportarse "
        "fraternalmente los unos con los otros."
    ),
    "Chinese": "人人生而自由,在尊严和权利上一律平等。他们赋有理性和良心,并应以兄弟关系的精神相对待。",
    "Japanese": (
        "すべての人間は、生まれながらにして自由であり、かつ、尊厳と権利とについて平等である。"
        "人間は、理性と良心とを授けられており、互いに同胞の精神をもって行動しなければならない。"
    ),
}

for language, sample in texts.items():
    print(language, measure(sample, font))
```

注意：

- 上述字体路径只是 Linux/Noto 环境示例；在其他系统复算时必须改为实际字体文件，并同时记录该文件的哈希值；
- `wcswidth` 是终端列宽近似，不是浏览器或印刷排版宽度；
- `getlength` 返回字体行进距离，不是字形墨迹面积；
- 字体 collection 的不同区域 face 在本例中对这些字符串给出相同或几乎相同的 advance，但字形形状可能不同；
- 真实页面实验还需固定版心、字号、行高、断行算法和显示分辨率；
- 对用户字符限制，应优先计扩展字素簇；对存储限制，应另外计实际编码字节。
